The cast
Three players. One negotiation.
Before we write a single line of code, meet the three entities in every REST API interaction.
💼
The Client
You — the Candidate
You initiate every conversation. You send demands, check the current offer, update your terms, and finally — accept. In code, the client is your browser, app, or script.
HTTP requests
🏢
The Server
HR Portal
HR receives your requests, validates them, acts on them, and sends a response. In code, the server is FastAPI, Express, Django — anything that listens for HTTP.
reads / writes
📄
The Store
The Offer Letter
The current state of negotiations lives here. HR reads from it and writes to it on your behalf. In code, it's a database — or in our case, a plain Python dict.

The offer lands.
You want better terms.

HR sends you an offer. ₹14L salary. 2 days WFH. No joining bonus. You open the letter and think — I can do better.

So you pick up your laptop (the client), type your counter-offer, and send it to the HR Portal (the server). HR logs it in their system (the store) and sends you a response.

That exchange — request, process, respond — is a REST API call. Every back-and-forth in your negotiation maps to one of 5 HTTP verbs.

Role Senior Engineer

WFH days / week 2 ← you want 4
Joining bonus ₹0 ← you want ₹1L
Status ⏳ negotiating

This is the current state in the store. Every request you send changes it.

The story in 5 verbs
Every negotiation move is an HTTP verb
You'll use each one in the interactive journey. Here's the plot.
POST

Submit your counter-offer

You send your terms to HR for the first time. A new negotiation record is created in the store.

"I'm looking for ₹18L base, 4 days WFH, and a ₹1L joining bonus."
GET

Check what HR has on file

You ask: "What are the current terms?" HR reads the store and tells you exactly what's recorded — nothing changes.

"What does your system show for my negotiation right now?"
PUT

Full renegotiation

You've reconsidered everything. You send a completely new set of terms — the old record is fully replaced.

"Forget my last offer. Here are my new terms in full."
PATCH

Just one thing changed

HR met you halfway on WFH. You only need to update the salary. Everything else stays exactly as it was.

"Keep everything the same — just bump the salary to ₹17L."
DELETE

Negotiation closed — you accepted

You've agreed on terms. The counter-offer record is removed from the store. Negotiation is over.

"I accept. You can close out the negotiation."
POST

Welcome aboard — employee record created

A brand new POST. The candidate row is gone. A fresh employee record is created. Same verb, new resource, new life chapter.

"Candidate deleted. Employee #1042 onboarded. Welcome to the team."
